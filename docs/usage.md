# Сценарии использования

Ниже приведены типовые сценарии, которые можно проверить на локальной среде.

## Сценарий 1: Играть офлайн (без backend)
1. Запустите приложение: `npm start`.
2. На экране логина нажмите «Играть офлайн».
3. Игра доступна локально. Прогресс хранится в локальном хранилище (AsyncStorage).

Примечание: AsyncStorage используется сторами (см. папку `store/`).

## Сценарий 2: Регистрация и синхронизация прогресса
1. Запустите backend на `http://localhost:5000`.
2. В приложении откройте экран логина (`app/login.tsx`).
3. Выберите режим «Регистрация героя», задайте ник и пароль.
4. После успешной регистрации токен сохраняется, а прогресс может быть отправлен на сервер (см. `AuthStore.authenticate`).
5. В табе «Лига героев» нажмите «Синхронизировать прогресс».

Ожидаемый результат:
- API возвращает профиль со значениями `coins/upgrades/stats`.
- Клиент применяет серверный снапшот в локальные сторы (`AuthStore.applyProfile`).

## Сценарий 3: Просмотр таблицы лидеров
1. Выполните вход.
2. Откройте таб «Лига героев» (`app/(tabs)/leaderboard.tsx`).
3. Нажмите «Обновить лигу».

Ожидаемый результат:
- Клиент вызывает `GET /api/leaderboard` и отображает список `entries`.

## Дополнительно: создание пользователя через seed-скрипт
Скрипт `server/seed.py` создаёт пользователя в интерактивном режиме.

```powershell
python -m server.seed
```

Примечание: `server/seed.py` создаёт одного пользователя и не заполняет дополнительные демо-данные.
